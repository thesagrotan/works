# Tailwind CSS Setup - Configuration Summary

## Overview
This document outlines the Tailwind CSS v4 configuration for the Responsive Portfolio Website project using Vite.

## Configuration Files

### 1. `tailwind.config.ts`
- **Location**: Root directory
- **Purpose**: Tailwind CSS configuration file
- **Content Paths**: Configured to scan `./index.html` and all files in `./src/**/*.{js,ts,jsx,tsx}`
- **Version**: Tailwind CSS v4.0.0

### 2. `postcss.config.js`
- **Location**: Root directory
- **Purpose**: PostCSS configuration
- **Plugins**:
  - `tailwindcss`: Process Tailwind directives
  - `autoprefixer`: Add vendor prefixes automatically

### 3. `vite.config.ts`
- **Tailwind Integration**: Using PostCSS (via `@tailwindcss/postcss`)
- **ESM Module**: Set `"type": "module"` in package.json for ESM compatibility
- **Plugin Registration**: Tailwind is processed through PostCSS, not as a Vite plugin

### 4. `src/styles/globals.css`
- **Primary Styles File**: Main CSS entry point
- **Tailwind Import**: `@import "tailwindcss";` at the top
- **Custom Properties**: Contains CSS custom properties for:
  - Color system (background, foreground, borders, etc.)
  - Typography variables
  - Dark mode variants
  - shadcn/ui theme variables

### 5. `src/main.tsx`
- **CSS Imports**:
  - `./styles/globals.css` (includes Tailwind)
  - `./styles/typography.css` (custom typography system)

## Package Dependencies

### Production Dependencies
- `tailwindcss`: ^4.0.0
- `tailwind-merge`: * (for merging Tailwind classes)
- `clsx`: * (for conditional class names)
- `class-variance-authority`: ^0.7.1 (for component variants)

### Development Dependencies
- `@tailwindcss/postcss`: ^4.1.16 (PostCSS plugin for Tailwind v4)
- `autoprefixer`: ^10.4.21 (for PostCSS browser prefixes)

## Tailwind CSS Features in Use

### 1. **Utility Classes**
The codebase uses Tailwind utility classes extensively:
- Layout: `flex`, `grid`, `block`, `inline-grid`
- Spacing: `gap-[Xpx]`, `mt-[Xpx]`, `mb-X`, `px-X`, `py-X`
- Sizing: `w-full`, `h-full`, `size-[Xpx]`, `max-w-[Xpx]`
- Colors: Custom color classes with CSS variables
- Typography: Custom font families and sizes
- Positioning: `absolute`, `relative`, `fixed`, `inset-0`
- Transforms: `rotate-[Xdeg]`, `scale-y-[X%]`
- Effects: `backdrop-blur-sm`, `shadow-[...]`
- Transitions: `transition`, `duration-300`

### 2. **Responsive Design**
- Mobile-first approach
- Breakpoints: `md:` (48rem/768px), `lg:` (64rem/1024px)
- Examples: `md:text-[24px]`, `lg:flex-row`, `lg:px-16`

### 3. **Custom Theme**
- CSS custom properties integrated with Tailwind
- Theme defined in `globals.css` using `@theme inline` directive
- Color variables: `--color-background`, `--color-foreground`, etc.
- Dark mode support via `.dark` class

### 4. **Advanced Features**
- Custom variants: `@custom-variant dark (&:is(.dark *))`
- Group hover: `group-hover:scale-105`, `group-hover:fill-stone-600`
- Hover states: `hover:bg-stone-200`, `hover:bg-stone-700`
- Grid utilities: `[grid-area:1_/_1]`
- Arbitrary values: `w-[346.68%]`, `top-[-1px]`

## Development Workflow

1. **Start Development Server**:
   ```bash
   npm run dev
   ```
   - Vite plugin watches for changes
   - Tailwind regenerates styles automatically

2. **Build for Production**:
   ```bash
   npm run build
   ```
   - Tailwind purges unused styles
   - Output optimized for production

## Important Notes

### Generated Files
- `src/index.css` is auto-generated by Tailwind during build
- **Do not edit** `src/index.css` manually
- This file is excluded in `.gitignore`

### Tailwind v4 Specifics
- Uses `@import "tailwindcss"` instead of `@tailwind` directives
- Integrated via PostCSS (`@tailwindcss/postcss`)
- Requires `"type": "module"` in package.json for ESM support
- No separate CLI needed
- Enhanced CSS-first configuration approach

### CSS Loading Order
1. `globals.css` - Tailwind base + custom properties
2. `typography.css` - Custom typography utilities

### shadcn/ui Integration
The project uses shadcn/ui components with Tailwind:
- Custom theme variables defined in `globals.css`
- Components in `src/components/ui/`
- Utilities: `cn()` from `utils.ts` for class merging
